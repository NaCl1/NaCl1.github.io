关键点

  • String.split(",")：把输入的字符串分成若干字段，比如 "id,name,age" → ["id", "name", "age"]。

  • Pattern.matches("\\W", token)：检查 整个 token 是否完全由单个“非单词字符”构成。

    • 在 Java 正则里：

      • \w = [A-Za-z0-9_]（字母、数字、下划线）

      • \W = [^A-Za-z0-9_]（非字母数字下划线）

    • 但是 ⚠️ Pattern.matches 要求 整个字符串都匹配，所以 "abc$" 不会返回 true，只有 "!" 这样的才会返回 true。

因此这里相当于是没有防护,

[[HQL注入]]

HQL 的一个有趣的点是它允许我们在查询中使用 new 关键字调用 Java 类的构造函数。对于大多数类，构造函数对类字段进行简单的赋值，因此它们根本没有用处。一个主要的例外是 JdiInitiator)。
文档
https://docs.oracle.com/en/java/javase/21/docs/api/jdk.jshell/jdk/jshell/execution/JdiInitiator.html#%3Cinit%3E%5C(int,java.util.List,java.lang.String,boolean,java.lang.String,int,java.util.Map%5C)

关于这个类的背景, 其与JShell有关:
JShell 本身并不会在当前 JVM直接执行你写的代码，它会：

  1. 启动一个新的“执行 JVM”（Execution VM），专门用来运行你输入的代码。

    • 这个 JVM 是用 JDI 启动的（相当于 IDE 调试器 attach 一个 JVM）。

  2. 把你的代码片段（snippet）编译成 class，通过 JDI 发送到那个 JVM 里。

  3. 在执行 JVM 里运行它，然后通过 JDI 把结果返回给 JShell。
    这个过程中，JdiInitiator 就是 负责第 1 步“启动并连接执行 JVM” 的类。

再比如:
当你在命令行运行 jshell，然后输入 System.out.println("hi") 时：

  • JdiInitiator 会启动一个子 JVM；

  • 通过 JDI 把你的代码片段（class 文件/字节码）送进去；

  • 然后拿回运行结果。

在启动JVM时, 会指定自定义的jvm启动参数, 对应构造器参数 customConnectorArgs
按照文档, 该参数是custom arguments passed to the connector, 这回传给connector
